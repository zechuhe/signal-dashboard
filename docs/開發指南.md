# Signal Dashboard 開發指南

## 版本: v2.0
## 更新日期: 2026-03-31

---

## 1. 新增事件分類

**場景**：需要新增一個事件類型，如「供應鏈」。

**步驟**：
1. 編輯 `config/app.json` → `categories`：
```json
"supply_chain": { "label": "供應鏈", "color": "#E17055", "marker": "▲▼" }
```
2. 完成。Filter bar、表單選擇器、圖表標記、事件列表全部自動生效。

**不需要改任何 JS/CSS。**

---

## 2. 新增市場（交易所）

**場景**：新增日本 TSE 市場。

**步驟**：
1. 編輯 `config/app.json` → `exchanges`：
```json
"TSE": { "label": "日股", "yahooSuffix": ".T", "currency": "JPY" }
```
2. 在 `data/stock_list.json` 加入 TSE 股票：
```json
{ "code": "7203", "name": "Toyota 豐田", "exchange": "TSE" }
```
3. 用 `scripts/fetch_prices.py` 拉取股價（需在腳本的 STOCKS dict 中加入對應條目）。

---

## 3. 新增數據源

**場景**：新增一個新的新聞來源 API。

**步驟**：
1. 編輯 `config/sources.json` → `sources` 加入：
```json
"new_source": {
  "name": "New Source Name",
  "enabled": true,
  "endpoints": { "api": "https://..." },
  "defaults": {}
}
```
2. 在 `scripts/fetch_news.py` 中新增對應的 `fetch_new_source()` 函數。
3. 管理面板（admin.js）會自動從 `sources.json` 讀取顯示。

---

## 4. 修改圖表參數

**場景**：修改 MA 均線週期、蠟燭顏色、VOL 高度。

**步驟**：
編輯 `config/app.json` → `chart`：
```json
{
  "chart": {
    "candle": { "upColor": "#FF0000", "downColor": "#00FF00" },
    "indicators": {
      "MA": { "periods": [5, 20, 60, 120] },
      "VOL": { "height": 100 }
    }
  }
}
```

**不需要改任何 JS。**

---

## 5. 修改 Port

**方法 A**：命令列 — `start.bat 9000`
**方法 B**：環境變數 — `set PORT=9000 && start.bat`
**方法 C**：config — 編輯 `config/app.json` → `app.port`（start.bat 未讀此值，需手動傳）

---

## 6. 新增前端功能模組

**步驟**：
1. 在 `js/` 下新建 `new_module.js`
2. 在 `index.html` 中 `app.js` 之前加一行 `<script src="js/new_module.js"></script>`
3. 模組中的函數可直接呼叫 `utils.js` 提供的工具（CATEGORIES / getChartConfig / getUserEvents 等）
4. 如需在 init 時執行，在 `app.js` 的 `init()` 中加入呼叫

**注意**：所有模組共用全域作用域。函數名不要與現有模組衝突。

## 7. 事件數據格式（完整欄位）

```json
{
  "event_id": "string (唯一 ID，user_ 開頭為用戶輸入)",
  "date": "YYYY-MM-DD (事件日期，用於圖表定位)",
  "date_received": "YYYY-MM-DD (消息取得日期，可選)",
  "date_event_estimated": "YYYY-MM-DD (推測事件發生日期，可選)",
  "title": "string (30 字內)",
  "description": "string (詳細描述)",
  "category": "string (對應 config/app.json categories 的 key)",
  "source_type": "private | news | government | company_ir | court_record | manual",
  "source_url": "string (來源 URL，私人消息留空)",
  "related_stocks": ["EXCHANGE:CODE"],
  "expected_impact": "positive | negative | neutral",
  "confidence": "number 0-1",
  "tags": ["string"],
  "metadata": {
    "person": "string (相關人物)",
    "person_level": "founder | c_suite | vp | director | manager",
    "company_market_cap_hkd": "number",
    "business_segment_ratio": "number 0-1",
    "deal_value_hkd": "number",
    "event_scale": "small | medium | large | landmark",
    "source_level": "board | c_suite | vp | manager | industry | internet"
  }
}
```

## 8. 股價數據格式

```json
{
  "meta": {
    "symbol": "EXCHANGE:CODE",
    "name": "string",
    "exchange": "string (config/app.json exchanges 的 key)",
    "currency": "string (HKD/USD/TWD/CNY)",
    "timezone": "string",
    "price_precision": "number (小數位數)"
  },
  "data": [{
    "date": "YYYY-MM-DD",
    "open": "number",
    "high": "number",
    "low": "number",
    "close": "number",
    "volume": "number (股)",
    "adj_close": "number (可選)",
    "prev_close": "number (可選)",
    "change": "number (可選)",
    "change_pct": "number (可選)"
  }]
}
```

**檔名規則**：`price_{exchange}_{code}.json`（如 `price_HKEX_0700.json`）
